  • Apache static no proxy needed at all (Apache handles `/cgi-bin` natively)
  • GitHub Pages no backend at all, data comes from the repo

// vite.config.ts
import { defineConfig, loadEnv } from 'vite'
import react from '@vitejs/plugin-react'
import { VitePWA } from 'vite-plugin-pwa'

export default defineConfig(({ mode }) => {
 const env = loadEnv(mode, process.cwd(), '')

 const isGithub = mode === 'github'
 const isApache = mode === 'apache'
 // mode === 'development' (npm dev) is the fallback

 const base = isGithub
 ? '/exercise/' // your GitHub Pages repo name
 : isApache
 ? '/sandboxexercise2/' // Apache subdir
 : '/' // npm dev (served at root by Vite)

 return {
 base,
 server: {
 port: 5173,
 host: true,
 // npm dev: proxy /cgi-bin to Apache (which runs the CGI backend)

Simplified deploy.sh

Since there's no longer any config file swapping, your `deploy.sh` becomes: 

npm run build:apache
cp -r dist/* /mnt/d/Apache/html/sandboxexercise2/
cp vite.config.ts  NOT NEEDED  --  just run build:apache

The key insight is that the `server.proxy` block only applies during `npm run dev` (Vite's dev server), so it's harmless to leave it in the config for all modes -- it simply never fires during a `build`. This means you don't need conditional proxy logic at all; just leave it pointed at Apache port 80 and it will only ever be used when you're running the dev server locally.
